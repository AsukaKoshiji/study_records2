# 要件定義書

## システム概要

学習記録を管理するWeb APIアプリケーションを作成する。

## 開発目的

学習内容をカレンダーに記録・目標設定・進捗確認できるようにする。

## 想定ユーザー

- 学習記録を継続したい人
- 学習進捗を可視化したい人
- 個人学習を管理したい人

## システム構成

|要素|内容|
|---|---|
|Frontend|未定（将来的にNext.js導入予定）|
|Backend|FastAPI|
|DB|MySQL|
|環境構築|Docker|

## 機能要件

### 学習記録機能

ユーザーは学習内容を記録できる。

#### 登録項目

- 学習タイトル
- 学習内容
- 学習時間
- 学習日
- メモ

#### 操作

- 新規登録
- 一覧表示
- 詳細表示
- 編集
- 削除

### 目標設定機能

ユーザーは学習目標を設定できる。

#### 目標単位

- 日次
- 月次

#### 管理内容

- 目標時間
- 進捗
- 達成状況

### カレンダー機能

カレンダー上で学習記録を確認できる。。

#### 表示内容

- 学習実施日
- 学習時間
- 達成状況

### 進捗確認機能

ユーザーは進捗を確認できる。

#### 確認内容

- 日次学習時間
- 月次学習時間
- 目標達成率
- 学習継続状況

## 非機能要件


### API
REST API形式で実装する

### DB
MySQLを使用する

### 開発環境
Dockerで起動できるようにする

### セキュリティ

- SQLインジェクション対策を行う
- ORMを使用する

### 保守性 

- レイヤードアーキテクチャを採用する
- APIごとに責務を分離する
- Dockerにより環境差異をなくす

## データ管理

- データ記録はMySQLで管理する
- 日付ごとに学習内容を保存する
- 学習時間は分単位で保持する
- 削除済みデータは物理削除する

## 用語定義

|用語|説明|
|---|---|
|学習記録|ユーザーが登録する学習内容|
|目標|ユーザーが設定する学習時間|
|進捗|目標に対する達成状況|

## 将来的な拡張

- AIによる学習分析
- 学習傾向の可視化
- 学習カテゴリ分類
- タグ機能
- 認証機能

- モバイル対応は将来的に対応予定
